  • Preserving, Caring and maintaining this code is essential to the life of the artwork

In addition to a physical vault there is also a digital art vault

Git (/ɡɪt/) is a version-control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source-code management in software development, but it can be used to keep track of changes in any set of files.

In addition to a Git we use a product call Bitbucket server which helps organize the source code and offers a clear user interface. Here we can review the source code

We can make branches or copies of the original source code when updates are required for conservation purposes. This allows us to safely test updates with out comprimising the original work

All changes are tracked with information on who made the change, a timestamp, descriptive comment and clear view of what was changed

Like provenance, there is an extensive record of of the history of the work. It is possible to revert back to any point in time.

  • Core Principles
  • Document original source code.
  • Branches represent particular features
  • Tag specific versions with exhibiton details.
  • Be atomic as possible with commits
